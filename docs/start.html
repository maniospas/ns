Get started with NS

Assignment safety

Use a bracket scope to prevent any assignment from leaking to enclosing scopes.

x = 0;
{
	x = x + 1;
	print(x);  //prints 1
};
print(x); //prints 0

Enclose code in a parentheses scope to transfer all assignments to the parent scope.

x = 0;
(
	x = x + 1;
	print(x);  //prints 1
);
print(x); //prints 1

Scope return value

Do not put a semicolon after the last expression to make it be the outcome of running the scope.

x = 0;
y = { //assignment safety
	x = x + 2; 
	x * x
};
print(y); //prints 4

new references the scope object itself. Return it and access its contents with the dot notation.

x = 0;
y = { //assignment safety
	z = x+1;
	new
};
print(y.z); //prints 1

Metaprogramming

Define formulas for pending evaluation with the := operator. This is more general than defining methods.

add(x,y) := {x+y}; // assign a formula
sub (z) from (x,y) := {x+y-z};
Point(x,y) := { // x, y in superscope
	elem sum := x+y; // no argument
	new // basically a constructor
}; 
print(add(1,2)); // prints 3
print(sub (add(1,2)) from (1,2)) // 0
print(Point(1,2).x) // prints 1 
print(Point(1,2).elem sum) // prints 3

The colon operator enters a scope from a foreign superscope while prioritizing the former's internal variable values. Use this to override any statement.

stricter = {
    base=super; //keep superscope
    (x)<(y) := { // changes super
        base: ( // run in base scope
			(x)<((y)-(1))
		) 
    };
    new
};
print(stricter:(1)<(2)); // modified
